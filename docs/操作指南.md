 # 分型面尖钢审查 Harness — 操作指南

 > 面向模具设计工程师 / NX 二次开发者
 > 目标：用 AI 自动完成分型面尖钢的检测→修复→验证闭环

 ## 一、这个工作流解决什么问题

 分型面尖钢是模具设计中常见的问题：分型面上某些区域钢位过薄，在注塑过程中容易断裂或变形。传统做法是工程师手动检查每个面，凭经验判断，效率低且容易遗漏。

 这个 harness 通过 AI 驱动的循环闭环，自动完成：
 1. 检测分型面尖钢区域
 2. 自动修复检测到的问题
 3. 重新检测验证修复效果
 4. 循环直到全部合格

 ## 二、环境要求

 | 组件 | 要求 |
 |------|------|
 | NX | 2306 版本 |
 | Visual Studio | 2022 (v143) |
 | Claude Code | 最新稳定版 |
 | 内网 VPN | 可访问 MCP 编译服务 |
 | playdll_v2.exe | 放置在 knowledge/standards/ |

 ## 三、快速开始

 ### 第一步：编译检测工具

 在 harness 目录下打开 Claude Code，运行：

 ```text
 /steel-review run E:\asd44\Documents\UI\考核\错误分型面1已带产品.prt
 ```

 AI 会自动检测是否有已编译的 DLL，如果没有则自动编译。

 ### 第二步：审查检测结果

 AI 读取诊断报告后会展示问题清单：
 - 哪些区域有尖钢
 - 严重程度（CRITICAL / WARNING / INFO）
 - 每个区域的位置和几何指标

 ### 第三步：启动修复循环

 ```text
 /steel-fix plan workspace/reports/diagnosis_xxx.json
 ```

 AI 会：
 1. 制定修复方案
 2. 生成修复 C++ 代码
 3. 编译修复 DLL
 4. 在 NX 中执行修复
 5. 自动重新检测
 6. 报告修复效果

 ### 第四步：循环直到完成

 如果有问题未解决，AI 会自动进入下一轮修复，直到：
 - 所有尖钢问题消除
 - 连续多轮无改善（报告人工）
 - 出现不可自动修复的问题

 ## 四、手动操作（当自动流程受阻时）

 ### 手动编译检测 DLL

 ```powershell
 cd E:\AAAA
 powershell -File scripts/compile-and-run.ps1 -Action compile-detect
 ```

 ### 手动执行检测

 ```powershell
 $env:STEEL_PRT_PATH = "E:\prts\test.prt"
 $env:STEEL_OUTPUT_PATH = "workspace/reports/diag.json"
 playdll_v2.exe knowledge/standards/detect_sharp_steel.dll
 ```

 ### 查看历史记录

 每次检测的报告保存在 `workspace/reports/` 目录下，按时间戳命名。

 ## 五、常见问题

 **Q: 检测 DLL 编译失败？**
 A: 检查 MCP 服务是否可连接，检查源码语法。

 **Q: playdll_v2.exe 找不到？**
 A: 从 mold-plugin-harness 的 knowledge/standards/ 目录复制。

 **Q: 检测结果不准确？**
 A: 调整 knowledge/standards/sharp_steel_criteria.md 中的阈值。

 **Q: 修复后问题不减反增？**
 A: 需要人工介入——修复可能引入新的尖钢区域，检查修复策略是否合适。

 ## 六、交付物

 最终输出的文件：
 - `workspace/reports/diagnosis_final.json` — 最终检测报告（PASS）
 - `workspace/fixed_result.prt` — 修复后的 PRT 文件
 - `workspace/reports/fix_report.json` — 修复过程记录
